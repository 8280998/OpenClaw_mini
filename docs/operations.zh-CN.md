# 运维说明

## 日常使用流程

1. 在 Mac mini 上启动机器人服务。
2. 发送 `/status` 确认服务可用。
3. 使用 `/backend` 和 `/mode` 确认当前规划后端与执行范围。
4. 使用 `/run` 执行明确的 shell 命令。
5. 使用 `/goal` 或直接发送自然语言文本，提交需要规划的高层目标。
6. 需要复用预设能力时，使用 `/skills` 和 `/skill`。
7. 使用 `/task <id>` 查看某个任务的持久化结果。

## 可观测性

每个任务都会保存以下信息：

- 发起请求的 chat ID
- 原始输入
- 当前状态
- 规划器输出
- 实际执行的命令
- stdout 和 stderr
- 时间戳

这些数据目前存放在 SQLite 中，对于单机部署已经足够。如果后续要支持多用户或更高吞吐，可以把同样的表结构迁移到 Postgres。

## 故障处理

规划失败时：

- 任务本身仍然会被记录
- 服务会把错误返回到 Telegram
- `/run` 直接执行模式仍可继续使用

命令执行失败时：

- 非零退出码会被保存
- stderr 会出现在 Telegram 摘要中
- 同一目标下后续命令默认停止执行

## 升级方向

最自然的下一步包括：

- 引入队列 worker，让 Telegram 处理更流畅
- 将部分输出流式回传到 Telegram
- 给被拦截命令加 inline 审批按钮
- 把日志或文件作为 Telegram 文档发送
- 支持远程 git checkout 和分支工作流

## Skill 运维

应用级 skill 是本地 JSON 预设。新增 skill 的方式是：

1. 把新的 `*.json` 文件放进配置的 skills 目录。
2. 文件中至少包含 `name`、`description` 和 `prompt_prefix`。
3. 重启机器人服务，或者未来再加热加载命令。
